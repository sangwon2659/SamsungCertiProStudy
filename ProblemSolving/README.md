1. 우선 순위 관리(PQ)
- Node들을 특정 기준에 맞춰 우선순위를 관리한다. (Ex. Price가 작은 순서, ID가 높은 순서로 Node관리)
- Set나 PriorityQueue를 사용한다면, 문제 쉽게 풀 수 있음
- 다만, 최근 경향은 단순히 STL로 찾는 것뿐만 아니라, 특정 값을 key로 탐색을 효율화 하는 등의 스킬이 필요한 경우가 많다
- H2220_성적조회 / H2506_온라인마트 / H2512_공항주차장 / H2519_카페2 / H2526_기계식주차장

2. 최단 경로
- 플로이드-워셜이나 다익스트라 같은 최단거리 알고리즘을 활용해서 문제를 해결하는 유형
- 예전 문제의 경우 단순 알고리즘 사용으로 풀리는 문제도 종종 출제되었지만, 단순 적용하는 유형은 지양되고 있는 추세
- 최단거리 알고리즘의 동작방식을 이해해야 풀이할 수 있도록 출제되는 중
- ㄱ) 중간에 탈출조건 또는 필수 방문 정점 추가
- ㄴ) Single-Source가 아닌 Multi-Source (커피점 & 제과점 문제)
- ㄷ) 다른 유형과 결합 (파일공유 문제)
- H2402_화물운송 / H2502_경유지운송 / H2507_전기차충전소 / H2515_통행료인상 / H2522_도로파괴

3. 구간 나눠서 풀기
- 구간 N이 주어지고, 구간별 Range Update / Range Query / Point Update / Point Query 등이 구현 요구된다
- Bucket, Segment Tree 등을 사용해서 효율적으로 값 업데이트 및 쿼리를 할 수 있어야 한다
- H2405_OnlineStreamer / H2511_음악방송

4. 구간 나워서 풀기 (2차원)
- 위 유형의 2차원 버전
- 2차원 공간에서의 탐색 범위를 줄이는 것이 포인트이며, 문제에서 9 <= N <= L * 30 등과 같이 범위로 힌트를 주는 경우도 있음 (이 경우에는 30 * 30으로 Bucket을 만들어 관리하라는 뜻)
- Union-Find 등과도 같이 연계되어 출제되고 있음 (광물 운송 문제)
- H2504_사각형그리기 / H2508_도로연결

5. Union Find
- Root를 찾고, 두 개의 집합을 하나로 합치는 문제유형, 여러 개의 그룹이 Merge가 빈번하게 발생하는 경우
- 여러 개의 Group이 있는 상황에서, 하나의 Group으로 Merge가 필요한 경우
- H2502_경유지운송 / H2518_광물운송

6. Time Base API 구현
- API에 mTimeStamp가 존재하며, API가 수행되기 전에 process(mTime) 등을 호출해서 작업들을 시간 Base로 처리하는 유형
- 보통 문제에 "mTime에 예정된 일이 있다면 완료된 후에 처리한다"와 같은 조건이 붙는다
- H2503_파일다운로드2 / H2520_지렁이게임
